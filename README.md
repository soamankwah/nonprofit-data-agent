# 🤖 Nonprofit Data Quality Agent
### Built with Claude API — by Samuel Amankwah

A Claude-powered AI agent that helps **nonprofit program staff** monitor their data, catch errors automatically, and ask questions about their own programs — without writing a single line of SQL.

Built as part of my application to the **Claude Corps Fellowship** (CodePath × Anthropic).

---

## 🎯 The Problem This Solves

Nonprofit program managers spend hours each week manually reviewing spreadsheets for errors — missing client records, duplicate entries, incorrect statuses, and broken KPIs. They often don't have a data analyst on staff and can't write SQL to investigate problems themselves.

This tool gives them:
- **Automated daily pipeline checks** that flag issues in plain English
- **A conversational interface** to query their data naturally ("Which clients missed two appointments this month?")
- **A summary report** they can share with leadership — generated by Claude, no analyst required

---

## 🏗️ Architecture

```
CSV / SQLite data  →  Pipeline Checker  →  Claude API  →  Plain-English Report
                                                ↑
                         Streamlit Chat UI  ────┘
                       (staff ask questions here)
```

**Stack:**
- `Python 3.10+`
- `Claude API` (claude-sonnet-4-6) — the brain
- `SQLite` — lightweight database, no setup required
- `Streamlit` — simple web UI for non-technical staff
- `Pandas` — data loading and validation

## 🚀 Quick Start

### 1. Clone the repo
```bash
git clone https://github.com/samuelamankwah/nonprofit-data-agent.git
cd nonprofit-data-agent
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Add your Anthropic API key
```bash
cp .env.example .env
# Open .env and add your key:
# ANTHROPIC_API_KEY=sk-ant-...
```

### 4. Load the sample data
```bash
python src/load_data.py
```

### 5. Run the pipeline check
```bash
python src/pipeline_checker.py
```

### 6. Launch the chat interface
```bash
streamlit run src/app.py
```

Then open `http://localhost:8501` in your browser.

## 💬 What Staff Can Ask

The chat interface understands plain English questions like:

> "Which clients have missed more than 2 appointments this month?"

> "Are there any duplicate client records?"

> "Show me a summary of program attendance for March."

> "Which case workers have the highest caseloads right now?"

> "Flag any clients whose status hasn't been updated in 30 days."

Claude translates the question into a database query, runs it, and explains the results in plain English — no SQL needed.

## 🔍 Pipeline Checks (Automated)

The checker (`src/pipeline_checker.py`) runs these validations and sends results to Claude for a plain-English summary:

- ✅ Missing required fields (name, ID, enrollment date)
- ✅ Duplicate client IDs
- ✅ Invalid status values
- ✅ Stale records (not updated in 30+ days)
- ✅ Appointments with no linked client
- ✅ KPI calculation errors in outcome metrics

**Example output:**
```
📋 DATA QUALITY REPORT — June 17, 2026

✅ 3 issues found across 50 client records:

1. MISSING DATA: Client IDs C-0023 and C-0041 are missing enrollment dates.
   → Action: Ask the intake team to complete these records before end of week.

2. DUPLICATE: Client "Maria Johnson" appears twice (IDs C-0011 and C-0034).
   → Action: Review with your case manager and merge or remove the duplicate.

3. STALE RECORD: Client C-0007 has not had a status update in 47 days.
   → Action: Check in with the assigned case worker to confirm current status.

Overall data health: 94% ✅
```

## 🧠 How the Claude Integration Works

```python
# src/agent.py (simplified)
import anthropic

client = anthropic.Anthropic()

def ask_about_data(user_question, db_schema, query_results):
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        system="""You are a helpful data assistant for nonprofit program staff.
        You explain data findings in plain, clear English — no jargon.
        When you find problems, you suggest specific, actionable next steps.
        Always be encouraging and assume the user is doing their best.""",
        messages=[{
            "role": "user",
            "content": f"""
            The user asked: {user_question}
            
            Database schema: {db_schema}
            Query results: {query_results}
            
            Please explain what this means in plain English and suggest what to do next.
            """
        }]
    )
    return response.content[0].text
```
